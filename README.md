# sales-sql-analysis
SQL analysis project exploring sales performance using filtering, sorting, aggregation, grouping, and conditional logic. Includes structured queries for revenue analysis, customer insights, product performance, and operational metrics. Ideal for data analytics and BI portfolios.

---

## 🗂️ Dataset Description

The project assumes a table named **`sales`** with fields such as:

- `Order_ID`
- `Order_Date`
- `Customer_Name`
- `City`
- `Category`
- `Product_Name`
- `Quantity`
- `Total_Amount`
- `Total_Amount`
- `Payment_Mode`
- `Rating`
- `Delivery_Status`

---

## 🧠 SQL Concepts Covered

### **1. Basic Queries**
- Selecting all columns  
- Selecting specific fields  
- Filtering with `WHERE`  
- Limiting results  

### **2. Sorting**
- Highest/lowest revenue  
- Sorting by quantity  
- Sorting by order ID  

### **3. Aggregations**
- `COUNT`, `SUM`, `AVG`, `MAX`, `MIN`  

### **4. Grouping**
- Revenue by category  
- Orders by city  
- Average rating by category  
- Quantity sold by product  

### **5. HAVING Clause**
- Filtering aggregated results  
- Identifying high‑revenue categories  
- Identifying low‑order cities  

### **6. Conditional Queries**
- Cancelled orders  
- Delivered orders with 5‑star ratings  
- High‑value card transactions
